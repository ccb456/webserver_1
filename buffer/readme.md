## 一、Buffer类的整体功能
这是一个典型的 **自动增长环形缓冲区封装**，目标是替代 C 风格的固定大小数组（比如 char buf[1024]），提高内存安全性与灵活性。

### Buffer的设计目标
存放 socket 读入的数据（recv）；

存放待发送的数据（send）；

自动扩容（防止溢出）；

提供简洁的读写接口；

避免频繁的内存拷贝。

### 设计亮点
✅ 内存安全：使用 std::vector<char> 自动管理内存，不使用裸指针。

✅ 自动扩容与前移：避免频繁分配与内存碎片。

✅ 高性能读写：使用 readv 系统调用，一次读多块。

✅ 线程安全性：Buffer 本身未加锁（因为每个连接独占一个 Buffer，不需要锁）。

✅ 易扩展：非常容易替换成环形缓冲区或支持零拷贝（sendfile）的实现。
